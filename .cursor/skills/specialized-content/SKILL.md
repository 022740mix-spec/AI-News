# 専門コンテンツ — 比較・リファレンス・ビジュアル

## 1. ツール比較記事の評価軸

### 客観性の原則

1. 評価軸を先に定義する（感覚ではなく基準で比較）
2. 料金・機能・制限は公式ドキュメントから取得
3. 「最高」「最強」は使わない（「〜の用途に向いている」と条件付き）
4. 時点を明示する（「2026年3月時点」）
5. 複数の視点で評価する（初心者/中級者/上級者、個人/企業、無料/有料）

### 共通評価軸（5段階・5軸 + 料金）

| 軸 | 何を見るか |
|----|-----------|
| 導入のしやすさ | 初回セットアップの手間、前提知識 |
| 学習曲線 | 使い始めてから実用レベルまでの時間 |
| 機能の幅 | 対応言語、統合機能、拡張性 |
| AI の質 | 生成コードの正確性、文脈理解、モデル選択肢 |
| 料金体系 | 無料枠、月額、従量課金（公式から引用） |
| 企業利用 | SSO、監査ログ、利用規約 |

### カテゴリ別の追加軸

**バイブコーディングツール**: エディタ統合、エージェント能力、コンテキスト管理、音声入力対応、チーム機能

**生成メディアツール**: 生成品質、商用利用ライセンス、カスタマイズ、API 提供

### レベル定義

| レベル | 特徴 |
|--------|------|
| 入門 | コード経験がほぼない、AIツール初体験 |
| 初級 | 簡単なコードは読める、AIチャットは使ったことがある |
| 中級 | 日常的にコードを書く、Git を使う |
| 上級 | 複雑なプロジェクトを管理、CI/CD を構築 |

おすすめの出し方: 「{レベル}で{目的}なら → {ツール名}（理由を1文）」

## 2. 比較記事の構成テンプレート

### 単発比較（A vs B）

```
title: 「Cursor vs VS Code + Copilot — どちらを選ぶか（2026年3月版）」
type: "feature"

body 構成:
1. リード — 両者の位置づけと、この比較が誰に役立つか
2. 共通点 — 差が出ない部分
3. 評価軸ごとの比較（表 + 各軸の解説段落）
4. レベル別おすすめ（入門/中級/上級 × 個人/企業）
5. 結論 — 「〜なら A、〜なら B」（条件付き）
```

### カテゴリ横断比較

```
title: 「【比較】バイブコーディングツール全体像 — 2026年春版」
type: "feature", category: "special"

body 構成:
1. リード — なぜ今この比較が必要か
2. レベル別ロードマップ
3. ツール一覧表（全評価軸を横並び）
4. カテゴリ別の深掘り（IDE系 / CLI系 / ブラウザ系）
5. 料金比較表
6. まとめ — 目的別の最短パス
```

### 更新サイクル

| トリガー | アクション |
|---------|-----------|
| メジャーバージョンアップ | 該当ツールの評価を更新 |
| 料金改定 | 料金比較表を更新 |
| 新ツール登場 | 比較対象に追加 |
| 四半期ごと | 全体比較記事の鮮度チェック |

> **ハンドオフ**: 比較記事は時点明記 + `lastReviewed` 更新が必須。`lastReviewed` の管理は `data-quality` スキルの管轄。更新後は必ず `lastReviewed` を当日に設定すること。

## 3. コマンドリファレンスの記法

### 対象ツール

| ツール | コマンド体系 |
|--------|-------------|
| Claude Code | `claude` CLI |
| Cursor | エディタ内コマンド + 設定ファイル |
| Codex | `codex` CLI |
| GitHub Copilot | エディタ拡張 + `gh copilot` |
| uv / Ruff | `uv` / `ruff` CLI |

### 機能別の分類

起動・セッション / 権限・モード / 設定ファイル / 入出力 / エージェント・自動化 / デバッグ・トラブルシュート

### コードブロックの原則

**1コマンド = 1コードブロック**（コピーしやすくする）:

```javascript
// 良い例
"**Auto モードで起動（権限スキップ）**:",
"```bash\nclaude --dangerously-skip-permissions\n```",

// 悪い例（複数コマンドが1ブロックに混在）
"```bash\nclaude\nclaude -c\nclaude -p \"...\"\n```",
```

例外: パイプやオプションの組み合わせなど、まとめてコピーすべきものは1ブロックにする。

### 初心者向けの配慮

- 各ツールの冒頭に「まずこれだけ（最初の5分）」セクション
- コマンドに `#` コメントで何が起きるかを添える
- 危険なコマンド（`--dangerously-skip-permissions` 等）にはなぜ危険か・いつ使うかを明記

### リファレンス記事の構成

```
title: 「【コマンド辞書】Claude Code — 起動・モード・設定の全コマンド」
type: "feature", category: "cli"

body 構成:
1. リード
2. まずこれだけ（最初の5分）
3. 起動・セッション
4. 権限・モード
5. 設定ファイル
6. 入出力
7. エージェント・自動化
8. トラブルシュート
```

### ツール横断の対照表

```javascript
tables: [{
  caption: "やりたいこと別コマンド対照表",
  headers: ["やりたいこと", "Claude Code", "Codex", "備考"],
  rows: [
    ["対話セッション開始", "claude", "codex", ""],
    ["前回の続き", "claude -c", "codex --continue", ""],
    ["権限スキップ", "claude --dangerously-skip-permissions", "codex --full-auto", "本番では非推奨"],
  ]
}]
```

## 4. UI/画像の仕様

### coverImage

```javascript
coverImage: {
  src: "articles/example.webp",  // public/articles/ に配置
  alt: "画像の説明（必須）",
  caption: "出典: ..."           // 任意
}
```

- フォーマット: WebP 推奨（JPEG/PNG も可）
- サイズ: 幅 1200px 以上、16:9 または 2:1 のアスペクト比
- ファイル名: `articles/{article-id}.webp`
- 権利: 公式素材・CC ライセンス・自作のみ使用可

### figures

```javascript
figures: [{
  src: "articles/diagram-example.svg",
  alt: "図の説明",
  caption: "キャプション（任意）",
  afterParagraph: 2  // body[2] の直後に配置（0始まり）
}]
```

- SVG 推奨（拡大縮小に強い）
- 1記事あたり最大3枚が目安
- `afterParagraph` は本文の段落数以内であること

### tables

```javascript
tables: [{
  afterParagraph: 1,
  caption: "主要モデルの比較",
  headers: ["モデル", "パラメータ", "コンテキスト長", "価格"],
  rows: [
    ["GPT-5.4 mini", "非公開", "128K", "$0.40/1M tokens"],
    ["Claude Opus 4.6", "非公開", "1M", "$15/1M tokens"]
  ]
}]
```

### charts

- `bars[].value` は全て `primarySources` で確認できる公式値のみ
- `unit` フィールドが正しいか（%, x, 万 等）
- `maxValue` がデータを見やすく表現しているか

## 5. 図表・チャートの配置ルール

- `afterParagraph` は参照先の段落のすぐ後に配置
- テーブルは横スクロール対応（`overflow-x: auto`）
- 画像は `max-width: 100%` で縮小
- ダークモードで視認性を検証する

### 企業ロゴ

- フォーマット: SVG 推奨（32x32px で表示）
- 配置先: `public/logos/{company-id}.svg`
- 公式プレスキットで配布されているもののみ使用
- ダークモードで見えなくなるロゴは避けるか CSS で対応

## 6. サイト全体のデザイン基準

### デザイントークン（index.css）

```css
/* 触ってよいトークン */
--font, --radius-sm/md/lg, --transition

/* テーマ切替 */
[data-theme="light"] { ... }
[data-theme="dark"] { ... }
```

### レスポンシブ対応

- モバイルファースト（min-width でブレークポイント）
- 画像は `max-width: 100%` で縮小
- テーブルは横スクロール対応

### OG 画像

- `og.svg`（`public/`）がデフォルト
- 記事固有は `coverImage.src` を使用
- Twitter Card: `summary_large_image`
