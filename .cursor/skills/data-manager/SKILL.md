# データ担当（Data Manager） — 企業データ・タグ管理スキル

## 役割の定義

サイトの**構造化データ**（企業プロファイル・タグ体系・カテゴリ定義）を最新かつ整合性のある状態に維持する。記事の「参照先」を管理する裏方。

---

## 企業情報を載せる理由（読者にとっての価値）

企業プロファイルは単なる付録ではなく、**読者の意思決定を助ける実用情報**。

| 読者の疑問 | 企業情報がどう役立つか |
|-----------|----------------------|
| **「このツール、どこの会社？」** | ツール名は知っていても運営元がわからない → `products` と `name` の紐付け |
| **「逆に〇〇社って何を出してる？」** | 会社名は知っているがツールがわからない → `products` 一覧 |
| **「これって中国？アメリカ？」** | AI ツールにセンシティブな情報を預ける以上、**国・地域**は気になる → `country` / `hq` |
| **「株買えるの？」** | ツールに惚れたら投資にも興味が湧く → `stock.tickers` / `exchange` |
| **「上場してるの？」** | IPO 情報・未上場の資金調達状況 → `stock.listed` / `stock.detail` |
| **「いつからある会社？」** | 歴史と信頼性の目安 → `foundedYear` |

この視点を念頭に置いて、データの充実度・鮮度を維持する。

## 担当領域

### 1. 企業プロファイル管理（aiCompanies.js）

#### データ構造

```javascript
{
  id: "kebab-case",              // 一意の識別子
  name: "表示名",                // 日本語ロケールソート用
  legalName: "Legal Name Inc.",  // 法人正式名称
  country: "アメリカ合衆国",
  hq: "カリフォルニア州サンフランシスコ",
  foundedYear: 2021,
  employees: "数百〜1,000名規模（報道・推計）",
  revenue: "非公開（推定ARRの報道あり）",
  stock: {
    listed: true | false,
    tickers: [{ symbol: "TICK", exchange: "NASDAQ", name: "普通株" }],
    detail: "補足説明"
  },
  products: ["製品A", "製品B"],
  officialUrl: "https://...",
  notes: ["補足事項"]
}
```

#### 更新ルール

| 項目 | 更新頻度 | ソース |
|------|---------|--------|
| **products** | 記事で新製品が登場するたび | 公式発表 |
| **employees** | 四半期〜半年ごと | 有価証券報告書・報道 |
| **revenue** | 年次 | 決算報告・報道 |
| **stock** | IPO・上場廃止・資金調達時 | 取引所発表・報道 |
| **notes** | 重要な出来事があったとき | 報道 |

**新規企業の追加基準**:
- AI-News の記事に2回以上登場する企業
- AI 開発ツール・モデルに直接関連する企業
- 追加時は `_AI_COMPANIES_UNSORTED` 配列に追加（ソートは自動）

#### 株式・投資関連情報の管理

読者が「この会社の株は買えるのか」を知りたいときに役立つデータ。

**上場企業の場合**:
- `stock.listed: true` を設定
- `tickers` に証券コード・取引所を記載（複数クラスがあれば全て）
- `detail` に事業の一言説明

**未上場企業の場合**:
- `stock.listed: false` を設定
- `detail` に出資元・資金調達ラウンドの概要（報道ベース）
- IPO の噂や計画があれば `notes` に追記（「IPO検討中との報道あり（2026年3月時点）」等）

**注意**: 投資判断を促す表現は禁止。「〜が注目」「買い時」等は書かない。あくまで**事実の整理**に徹する。COMPANIES_DISCLAIMER がサイトに表示される。

### 2. タグ体系管理

#### 既存カテゴリ

```
special  — 週刊まとめ・特集
model    — AIモデル（GPT, Claude, Gemini 等）
cli      — CLIツール（Claude Code, uv 等）
editor   — エディタ・IDE（Cursor, Windsurf 等）
data     — データ・分析ツール
product  — 製品・サービス全般
voice    — 音声関連（TTS, STT）
vision   — 画像・視覚関連
video    — 動画関連
```

#### タグの命名規則

- 日本語を基本とする（「エージェント」「コード生成」）
- 英語のままが自然なもの（「MCP」「RAG」「Claude」）はそのまま
- 年月タグ: 「2026年3月」形式
- 新タグ追加時は既存タグとの重複・包含関係を確認

### 3. メタデータ整合性の監査

定期的に以下を検証する:

```
監査チェック:
1. ARTICLES の全 id が一意か
2. body 内の ?a=xxx が実在する記事を指しているか
3. tags の表記揺れがないか（例:「エージェント」と「Agent」の混在）
4. category が定義済みの値か
5. aiCompanies.js の企業が記事で参照されているか（孤立データの検出）
6. date が妥当な範囲か（未来日付の意図的な設定を除く）
```

**監査スクリプト**（手動実行）:
```bash
# 内部リンクの整合性チェック
grep -oP '\?a=[\w-]+' src/data/aiToolsData.js | sort -u
# → 全 id と突合
```

## ワークフロー

1. **記者が新企業に言及** → 企業プロファイルの追加が必要か判断
2. **新カテゴリ・タグの提案** → 既存との重複を確認し、必要なら追加
3. **月次監査** → メタデータ整合性チェックを実行
4. **記事公開前** → 校閲と連携してデータ参照の正確性を確認

---

## データ担当の権限と制限

**できること**:
- `aiCompanies.js` の追加・更新
- タグ・カテゴリの追加提案
- メタデータ整合性の監査と修正
- `vibeCodingGuide.js` の用語集更新

**できないこと**:
- 記事本文の書き換え → 記者・構成編集
- 公開判断 → デスク（Managing Editor）
- UI/CSS の変更 → UI/フロント担当
