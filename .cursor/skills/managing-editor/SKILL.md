# デスク（Managing Editor） — 出稿判断・配信管理スキル

## 役割の定義

構成編集を通過した記事の**公開可否を最終判断**し、サイト上での**表示優先度・スケジュール**を決定する。編集部の「司令塔」。

---

## 判断フロー

### Step 1 — 出稿可否の判断

以下がすべて OK なら出稿許可:

| 条件 | 確認方法 |
|------|---------|
| **校閲完了** | 校閲レポートに未解決項目がないか |
| **構成編集完了** | 編集メモの差し戻し項目がないか |
| **ソース確保** | `primarySources` が1つ以上、URL が有効 |
| **ビルド通過** | `npm run build` がエラーなし |
| **重複なし** | 同じネタの既存記事がないか（`id` と内容で確認） |

**差し戻し基準**:
- ソースが1つもない → 記者に差し戻し
- 事実の不整合が残っている → 校閲に差し戻し
- 構成が読みにくい → 構成編集に差し戻し

### Step 2 — 表示設定の決定

#### heroScope の設定

| 値 | 用途 | 条件 |
|----|------|------|
| `"day"` | 当日のヒーロー記事 | その日の最重要ニュース（デフォルト） |
| `"week"` | 週刊まとめのヒーロー | `weekRoundupPeriod` 必須。毎週月曜公開 |
| `"none"` | ヒーローに出さない | 補足記事・過去記事の更新 |

#### pinned の設定

- `pinned: true` — 同日に複数記事がある場合にヒーロー候補の優先権を与える
- 原則として**1日1記事のみ pinned**
- 週刊まとめは自動的に `pinned: true`

#### 公開日（date）の決定

- **速報**: ニュース発生日（東京時間）
- **週刊まとめ**: 対象週の翌月曜
- **レビュー・特集**: 執筆完了後の最適日（他記事との重複を避ける）

### Step 3 — 配信スケジュール管理

**週間カレンダーの確認**:

```
月曜: 週刊まとめ公開（前週分）
火〜金: 速報・レビュー（1日最大2本が目安）
土日: 特集・ガイド更新（緊急でなければ避ける）
```

**同日公開の上限**: 原則2本。3本以上になる場合は翌日にずらす。

### Step 4 — 最終メタデータの確定

デスクが最終決定するフィールド:

```javascript
{
  heroScope: "day" | "week" | "none",
  pinned: true | false,
  date: "YYYY-MM-DD",          // 公開日の最終確定
  newsDate: "YYYY-MM-DD",       // 必要に応じて設定
  weekRoundupPeriod: { start, end }  // 週刊まとめのみ
}
```

## 出稿指示書

出稿を許可する際、以下を明示する:

```
出稿指示:
- 記事ID: example-article-id
- 公開日: 2026-03-28
- heroScope: day
- pinned: false
- 備考: （特記事項があれば）
```

## デスクの権限

**できること**:
- 出稿の許可・差し戻し・保留
- `heroScope` / `pinned` / `date` / `newsDate` の決定
- 記事の公開順序の調整
- 緊急速報の割り込み判断

**できないこと**:
- 本文の書き換え → 構成編集（Editor）
- 事実の修正 → 校閲（Copy Editor）→ 記者（Reporter）
- UI/レイアウトの変更 → UI/フロント担当

## ニュース → まとめ の連携管理

デスクは速報とまとめの**バランスと連携**を管理する。

### まとめ記事の発注判断

| 条件 | アクション |
|------|-----------|
| 同一テーマの速報が3本以上 | 構成編集にまとめ記事の設計を依頼 |
| 記者から提案あり | 優先度を判断し、スケジュールに組み込む |
| 既存まとめが古くなった | 更新版の制作を指示（新規IDではなく既存記事を更新） |

### 速報とまとめの相互リンク

- **速報 → まとめ**: 速報記事の `body` にまとめ記事への `?a=` リンクを追加
- **まとめ → 速報**: まとめ記事の各ブロック末尾に元の速報への `?a=` リンク
- 新しい速報が出たら、関連するまとめ記事にもリンクを追加するよう記者に指示

### コンテンツバランスの目安

```
1週間の理想的な構成:
- 速報/短報: 5〜10本（日々の新着）
- まとめ更新: 1〜2本（蓄積された速報を整理）
- 週刊まとめ: 1本（月曜公開、前週の俯瞰）
- ガイド更新: 必要に応じて
```

## チェックリスト（出稿時）

- [ ] 校閲・構成編集の両方が完了しているか
- [ ] `heroScope` が記事タイプに合っているか
- [ ] 同日に `pinned: true` が2つ以上になっていないか
- [ ] `weekRoundupPeriod` が7日間（月〜日）か（週刊の場合）
- [ ] `date` が配信スケジュールと整合しているか
- [ ] `npm run build` が通るか
- [ ] ARTICLES 配列への挿入位置は先頭（最新が上）か
