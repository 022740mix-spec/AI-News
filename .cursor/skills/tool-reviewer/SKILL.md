# ツール評価（Tool Reviewer） — 横断比較・おすすめスキル

## 役割の定義

個々のニュースや製品レビューとは別に、**複数ツールを横断的・客観的に比較**し、読者のレベルや目的に応じた**おすすめ**を提示する。第三者的な視点で評価し、特定企業に偏らない。

---

## 評価の原則

### 客観性を保つためのルール

1. **評価軸を先に定義** — 感覚ではなく、明示した基準で比較する
2. **公式情報ベース** — 料金・機能・制限は公式ドキュメントから取得
3. **「最高」「最強」は使わない** — 「〜の用途に向いている」「〜を重視するなら」と条件付きで表現
4. **変化を前提にする** — 「2026年3月時点」のように時点を明示。ツールは日々変わる
5. **複数の視点** — 初心者/中級者/上級者、個人/企業、無料/有料で評価が変わることを示す

---

## 評価軸の体系

### 共通評価軸（全ツール共通）

| 軸 | 何を見るか | 評価方法 |
|----|-----------|---------|
| **導入のしやすさ** | 初回セットアップの手間、前提知識 | ステップ数、必要な事前知識 |
| **学習曲線** | 使い始めてから実用レベルまでの時間 | 初心者が最初の成果物を出すまでの想定時間 |
| **機能の幅** | できることの範囲 | 対応言語、統合機能、拡張性 |
| **AI の質** | 生成コードの正確性、文脈理解 | モデル選択肢、コンテキスト長、ツール連携 |
| **料金体系** | 無料枠、月額、従量課金 | 具体的な金額と制限（公式から引用） |
| **企業利用** | セキュリティ、ガバナンス、契約 | SSO、監査ログ、利用規約 |

### カテゴリ別の追加評価軸

#### バイブコーディングツール（IDE・AI アシスタント）

| 軸 | 何を見るか |
|----|-----------|
| **エディタ統合** | ファイル操作、ターミナル、デバッグとの連携 |
| **エージェント能力** | 自律的なタスク遂行、マルチステップ操作 |
| **コンテキスト管理** | プロジェクト全体の理解、ルールファイル対応 |
| **音声入力対応** | 音声での指示、ディクテーション |
| **チーム機能** | 共有設定、コードレビュー連携 |

#### 生成メディアツール（画像・動画・音楽）

| 軸 | 何を見るか |
|----|-----------|
| **生成品質** | 解像度、一貫性、プロンプト忠実度 |
| **商用利用** | ライセンス、著作権、補償 |
| **カスタマイズ** | スタイル調整、ControlNet 等 |
| **API 提供** | プログラムからの利用可否 |

## レベル別おすすめの設計

### レベル定義

| レベル | 特徴 | 前提知識 |
|--------|------|---------|
| **入門** | コードを書いた経験がほぼない、またはAIツール初体験 | ブラウザ操作ができる |
| **初級** | 簡単なコードは読める、AIチャットは使ったことがある | 基本的なプログラミング概念 |
| **中級** | 日常的にコードを書く、Git を使う | エディタ操作、CLI の基本 |
| **上級** | 複雑なプロジェクトを管理、CI/CD を構築 | アーキテクチャ設計、DevOps |

### おすすめの出し方

```
フォーマット:
「{レベル}で{目的}なら → {ツール名}（理由を1文）」

例:
入門でバイブコーディングを始めるなら → Claude チャット（セットアップ不要、対話だけで始められる）
中級でエディタ内 AI を使いたいなら → Cursor（AI ネイティブ設計で学習コストが低い）
企業で統制を重視するなら → GitHub Copilot（SSO・監査ログ・既存契約との親和性）
```

記事内の `tables` フィールドで表現する:

```javascript
  }
]
```

## 比較記事の構成テンプレート

### 単発比較（A vs B）

```
title: 「Cursor vs VS Code + Copilot — どちらを選ぶか（2026年3月版）」
type: "feature"
category: "editor"

body 構成:
1. リード — 両者の位置づけと、この比較が誰に役立つか
2. 共通点 — 両方にあるもの（差が出ない部分）
3. 評価軸ごとの比較（表で一覧 + 各軸の解説段落）
4. レベル別おすすめ（入門/中級/上級 × 個人/企業）
5. 結論 — 「〜なら A、〜なら B」（断定ではなく条件付き）
```

### カテゴリ横断比較

```
title: 「【比較】バイブコーディングツール全体像 — 2026年春版」
type: "feature"
category: "special"

body 構成:
1. リード — なぜ今この比較が必要か（ツールの変化が激しい）
2. レベル別ロードマップ（入門→初級→中級→上級の推奨パス）
3. ツール一覧表（全評価軸を横並び）
4. カテゴリ別の深掘り（IDE系 / CLI系 / ブラウザ系）
5. 料金比較表
6. まとめ — 目的別の最短パス
```

### おすすめ記事（「何から始める？」系）

```
title: 「バイブコーディング、何から始める？ — 2026年版おすすめツール」
type: "feature"
category: "special"

body 構成:
1. リード — 「選択肢が多すぎて迷う」問題の提示
2. 3つの道（個人初心者 / 個人開発者 / 企業）を提示
3. 各道ごとの推奨ステップ（段階表）
4. よくある質問（「Cursor と VS Code どっち？」等）
5. 注意点 — 料金・利用規約の変化に注意
```

## vibeCodingGuide.js との連携

サイトには既に以下のデータがある:
- `VIBE_PROGRESSION_PATH` — 段階的な道筋（①〜⑥）
- `VIBE_TOOL_COMBO_TABLE` — 組み合わせ表
- `VIBE_CODING_WHY_CLAUDE_FIRST` — まず Claude を勧める理由

**ツール評価記事との使い分け**:
- `vibeCodingGuide.js` = **ガイドタブ**の常設コンテンツ（概要レベル）
- ツール評価記事 = **記事タブ**の feature 記事（詳細な比較・時点付き評価）
- ガイドから記事へ `?a=` リンクで導線を張る

## 更新サイクル

ツール比較記事は**鮮度が命**。以下のタイミングで更新を検討:

| トリガー | アクション |
|---------|-----------|
| メジャーバージョンアップ | 該当ツールの評価を更新 |
| 料金改定 | 料金比較表を更新 |
| 新ツール登場 | 比較対象に追加 |
| 四半期ごと | 全体比較記事の鮮度チェック |

## チェックリスト（公開前）
