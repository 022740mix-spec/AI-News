# コマンドリファレンス — CLI 辞書・早見表スキル

## 役割の定義

AI 開発ツールの **CLI コマンド・起動オプション・設定ファイルの書き方** を、初心者にもわかるよう**ツール別・機能別に体系化**する。読者がすぐにコピー＆ペーストできる形で提供する。

---

## なぜこの記事が必要か

- `claude --dangerously-skip-permissions` のようなコマンドは**最初にやらないと使えない**が、初心者には発見しにくい
- ツールごとにコマンド体系が違う（Claude Code / Cursor / Codex）
- 公式ドキュメントは英語で散在しており、日本語で一箇所にまとまった早見がない
- 記事に文章で埋め込むと見つけにくい → **コードブロックで独立させ、コピーできる** 形が必要

|--------|--------|-------------|

各ツール内で以下のカテゴリに整理する:

| カテゴリ | 内容の例 |
|---------|---------|
| **起動・セッション** | 初回起動、セッション継続、引数付き起動 |
| **権限・モード** | パーミッションモード、自動承認、サンドボックス |
| **設定ファイル** | CLAUDE.md、.cursor/rules、スキルファイル |
| **入出力** | パイプ入力、ファイル指定、出力形式 |
| **エージェント・自動化** | Auto モード、サブエージェント、バッチ実行 |
| **デバッグ・トラブルシュート** | ログ確認、リセット、バージョン確認 |

### コードブロックの記法

記事の `body` 配列で、` ``` ` で始まる段落はコピーボタン付きコードブロックとしてレンダリングされる。

```javascript
body: [
  "Claude Code の起動コマンド一覧。",
  "**基本起動（対話モード）**:",
  "```bash\nclaude\n```",
  "**最初のメッセージ付きで起動**:",
  "```bash\nclaude \"このプロジェクトを説明して\"\n```",
  "**前回のセッションを再開**:",
  "```bash\nclaude -c\n```",
]
```

### 1コマンド = 1コードブロック の原則

**良い例**（コピーしやすい）:
```javascript
"**Auto モードで起動（権限スキップ）**:",
"```bash\nclaude --dangerously-skip-permissions\n```",
"```bash\nclaude -p \"タスク内容\" --allowedTools \"Edit,Write,Bash\"\n```",
```

**悪い例**（複数コマンドが1ブロックに混在）:
```javascript
"```bash\nclaude\nclaude -c\nclaude -p \"...\"\nclaude --dangerously-skip-permissions\n```",
```

→ 読者は1つだけコピーしたいのに全部コピーされてしまう。

### 例外: 複数行で1セットのコマンド

パイプやオプションの組み合わせなど、**まとめてコピーすべきもの**は1ブロックにする:
```javascript
"```bash\ncat spec.md | claude -p \"この仕様に沿ってテストを書いて\" \\\n  --allowedTools \"Edit,Write\" \\\n  --output-format stream-json\n```",
```

## リファレンス記事の構成テンプレート

### 単一ツールのコマンド辞書

```
title: 「【コマンド辞書】Claude Code — 起動・モード・設定の全コマンド」
type: "feature"
category: "cli"
tags: ["Claude Code", "CLI", "コマンドリファレンス"]

body 構成:
1. リード — 「claude --help より実用的な日本語早見」
2. 起動・セッション（基本の始め方）
3. 権限・モード（初心者が最初に必要なもの）
4. 設定ファイル（CLAUDE.md、スキル）
5. 入出力（パイプ、ファイル指定）
6. エージェント・自動化（Auto、サブエージェント）
7. トラブルシュート（ログ、リセット）
```

### ツール横断のコマンド比較

```
title: 「同じことをしたい — Claude Code / Cursor / Codex コマンド対照表」
type: "feature"
category: "special"
tags: ["コマンドリファレンス", "比較"]

body 構成:
1. リード — 「やりたいこと起点で引ける対照表」
2. 表: やりたいこと × ツール別コマンド（tables フィールド使用）
3. ツールごとの補足（違いの解説）
```

tables の例:
```javascript
tables: [
  {
    afterParagraph: 1,
    caption: "やりたいこと別コマンド対照表",
    headers: ["やりたいこと", "Claude Code", "Codex", "備考"],
    rows: [
      ["対話セッション開始", "claude", "codex", ""],
      ["前回の続き", "claude -c", "codex --continue", ""],
      ["権限スキップ", "claude --dangerously-skip-permissions", "codex --full-auto", "注意: 本番では非推奨"],
      ["ファ��ル指定実行", "cat file | claude -p \"...\"", "codex -f file \"...\"", ""],
    ]
  }
]
```

## 初心者向けの配慮

### 「まずこれだけ」セクション

各ツールのリファレンスの冒頭に、**最初の5分で必要なコマンドだけ**を抜粋:

```javascript
"**まずこれだけ（最初の5分）**:",
"```bash\n# 1. インストール確認\nclaude --version\n```",
"```bash\n# 2. 対話を始める（プロジェクトのルートで）\nclaude\n```",
"```bash\n# 3. 終了\n/exit\n```",
```

### コメント付きコマンド

コマンドだけでなく、**何が起きるか**を `#` コメントで添える:

```javascript
"```bash\n# 権限の確認なしで自動実行（開発中のみ推奨）\nclaude --dangerously-skip-permissions\n```",
```

### 危険なコマンドの警告

`--dangerously-skip-permissions` 等は本文で **なぜ危険か・いつ使うか** を明記:

```javascript
"**--dangerously-skip-permissions** は全ツール実行を自動承認する。開発環境でのみ使い、本番リポジトリでは避ける。",
```
