# 週刊ニュースまとめ — 取材・プラン・執筆スキル

## 調査メモ（海外サイトで共通する型）

公開テンプレートの断片・看板記事から、週まとめ系では次のパターンが繰り返し見られる。

| 出典の傾向 | 読み取れる作法 |
|------------|----------------|
| **CNN** などの週まとめ動画・政治デスク | 見出しが**週の両端の対比**（「週初めは〇〇、週末は△△」）で**物語の弧**を示す。デックは**一文の中で複数事象を「and / as」で接続**し、映像がなくても**時系列の筋**が通る。 |
| **Reuters**（Breakingviews「The week in …」、Morning Bid の week review 等） | **冒頭フック**＋**テーマ別の塊**（市場・政策・企業）。「今週学んだ五つのこと」のように**読者の利得**を明示する型も。 |
| **NYTimes**（The Morning 再設計の説明、各号のリード） | 複数トピックを**短いリード文で束ね**、本文では**一段ずつ具体**へ。ニュースレターは**挨拶や運用説明を前面に出さない**（編集方針は別レイヤ）。 |
| **共通（プロ・向け一般）** | **主語＋動詞＋帰結**は維持しつつ、**接続詞・従属接続**で「同時」「対照」「因果」「前後」を示す。**同じ日の出来事は一段落に束ね**、機械的な「A が B。C が D。」の繰り返しを避ける。 |

参照した公開例の例（調査時点）:

- CNN: [Trump starts week with State of the Union and ends with an unexpected meeting…](https://www.cnn.com/2026/02/27/politics/video/kc-feb27-digvid) — 週の**対比型**ヘッドラインと、複数出来事を**一続きのリード**で要約。
- Reuters: [The Week in Breakingviews](https://www.reuters.com/commentary/breakingviews/) — **週単位の論旨の束**。
- NYT Open: [A New Look for ‘The Morning’ Newsletter](https://open.nytimes.com/a-new-look-for-the-morning-newsletter-36bb135b5c79) — **読者向けに運用メタを前面に出さない**方向性。

※ URL は変わる可能性があるため、**型の説明**を優先し、リンクは補助とする。

## 執筆プラン（毎回この順で実行）

### Phase 1 — 編集条件の固定（読者に書かない）

- 対象**の7日間**（例: 月曜0時〜日曜23:59 JST）を `weekRoundupPeriod` とタイトルで一致させる。
- **掲載基準・配信時刻・「東京日付で限定」**などは、記事本文・概要に**書かない**（データファイル先頭コメントや編集メモにのみ）。

### Phase 2 — ファクトの束ね方

- カレンダー順に並べるが、**同じ日は1パラグラフに統合**する。
- **なぜ隣り合うか**（時系列／同時進行／因果／対照）を一文で示すと読みやすい。

### Phase 3 — 概要（excerpt / デック）

- **事実の要約**のみ。**感想・「一本の流れ」・「以下は日付順」は禁止**。
- **「今週、」で始まる短文の列挙だけ**になっていないか検査する。必要なら**「翌日」「同じ木曜には」「金曜には」「週末にかけて」**などで**時系列・対比**を示す**接続**を入れる（全部が同型接続にならないようバリエーションを付ける）。

### Phase 4 — 本文

- 各日: **リード（その日の核）→ 補足 → 内部記事リンク `?a=…`** の順が扱いやすい。
- **接続詞・副詞句**で関係を明示する（例）:
  - 時系列: **翌日、前日の審問を受けて、その数時間後に、週末にかけて**
  - 対照・並行: **一方で、同じ日には、法廷では、製品面では、開発者向けには**
  - 追加: **さらに、加えて**（**連続3回禁止** — 書き換える）
- **禁止**: 「一言で言うと」「静かな出だし」「全体としては」「以下は〜をまとめる」など**執筆メタ**。
- **禁止**: **編集ルール**や**運用の説明**の公開。

### Phase 5 — 品質チェック（推敲リスト）

- [ ] 概要は**誰が何をした／何が起きたか**だけか（感想・構造説明がないか）。
- [ ] **同じ接続詞**が段落頭に3回以上続いていないか。
- [ ] **同じ日**の段落で、無関係な二文が並んでいないか（並行・対照なら **一方で／その一方で** 等でつないだか）。
- [ ] 内部リンクは**深掘りの入口**として意味があるか。
- [ ] `npm run build`（AI-News 変更時）

## 悪い例 → 良い例（日本語）

**悪い（列挙・無接続）**

> 今週、A が X をした。B が Y をした。C が Z をした。

**良い（関係が見える）**

> A は水曜に X を公開した。**翌日の木曜**には B が Y を発表し、**同じ日に** C は Z を掲げたため、モデルとツールの告知が重なった。

**悪い（執筆メタ）**

> 以下は日付順に、つながりと入口をまとめる。

**良い（事実）**

> （メタ文は削除。いきなり最初の日の出来事から書く。）
