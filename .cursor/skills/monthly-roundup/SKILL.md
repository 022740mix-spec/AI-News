# 月刊ニュースまとめ — 構成・執筆スキル

## 週刊との違い（これが最重要）

| 観点 | 週刊 | 月刊 |
|------|------|------|
| 目的 | 速報のキャッチアップ | 全体像と趨勢の把握 |
| 構造 | 日付順 | テーマ別（3-5本の柱） |
| 粒度 | 個別ニュース | パターン・転換点 |
| 読者心理 | 「取り残されたくない」 | 「全体像を掴みたい」 |
| 文体 | 事実中心 | 事実 + 分析（意見ではなく解釈） |

**月刊は「4週分のホチキス止め」ではない。** 個別ニュースを繋いで「今月の転換点は何か」を示す物語にする。

## 執筆プラン

### Phase 1 — 月の振り返りと柱の抽出

1. その月に公開した全記事を読み返す
2. **3-5本のテーマ（柱）** を抽出する。例:
   - エージェント機能の競争
   - M&A と業界再編
   - モデル動向
   - 規制・法務
   - 来月の注目点
3. 各柱に**「なぜこれが転換点なのか」** を1文で書く

### Phase 2 — 各テーマの執筆

各テーマ（柱）のパラグラフは以下の構造:

```
テーマの要約（1文）
→ 具体的な出来事（事実、日付、数字）
→ 内部記事リンク [→ 詳細](?a=xxx)
→ なぜ重要か / 開発者への影響（1-2文）
```

- **個別記事の繰り返しにしない**。月全体を通じて見えた「方向性」を示す
- 具体的な数字（料金変更、ベンチマークスコア等）は入れるが、記事本文の詳細はリンクに任せる
- 内部リンクを積極的に使い、詳細記事への導線にする

### Phase 3 — 冒頭と締め

**冒頭**: 月全体の1文要約 + その月を特徴づけるキーワード3-5個
**締め**: 来月の注目点（予測ではなく「何を追うべきか」の示唆）

### Phase 4 — 品質チェック

- [ ] **各テーマに「なぜ重要か / 何が変わるか」があるか**
- [ ] **週刊の繰り返しになっていないか**。月間ならではの俯瞰的視点があるか
- [ ] **内部記事リンクが十分か**。各テーマに2-3本の関連記事リンク
- [ ] **数字と事実は1次情報ベースか**。推測や「〜とみられる」を排除
- [ ] **来月の展望は予測ではなく追跡ポイントか**
- [ ] AI生成っぽい均質な文体になっていないか
- [ ] `npm run build`

## 悪い例 → 良い例

**悪い（ホチキス止め）**

> 3月第1週は A があった。第2週は B があった。第3週は C があった。第4週は D があった。

**良い（趨勢の抽出）**

> 3月はエージェント機能の競争が一段と激化した月だった。Cursor 2.4 の並列サブエージェント、Claude Code の Auto mode 拡大、Gemini CLI の Plan Mode 追加が月末に集中し、各社が「AI に丸投げ」できる範囲を広げる方向で競った。

**悪い（事実の羅列）**

> Windsurf が値上げした。Cursor が Windsurf を買収した。OpenAI が Astral を買収した。

**良い（パターンの抽出）**

> M&A と価格改定が同時に進んだ。Cursor による Windsurf 買収で AI エディタの選択肢が減り、Windsurf の $15→$20 値上げで価格差も消滅。OpenAI は Astral（uv・Ruff）を買収し、Codex のパイプラインを強化。ツール企業の統合が「機能」から「エコシステム」の競争に移行している。

## データファイル上の設定

```javascript
{
  "id": "monthly-roundup-YYYY-MM",
  "type": "feature",
  "category": "special",
  "title": "【月刊まとめ】YYYY年M月の AI 開発ツール動向 — サブタイトル",
  "heroScope": "none",
  // 週刊の weekRoundupPeriod は不要
}
```

## プロジェクトでの適用場所

- 月刊データ: `src/data/aiToolsData.js`（`type: "feature"`, `category: "special"`）
- 公開タイミング: 月末〜翌月1日
- 本スキルは**月刊記事の新規作成・推敲時に必ず読む**
