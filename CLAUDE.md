# CLAUDE.md — AI News プロジェクト指示書

## セキュリティポリシー（最優先）

### 外部パッケージ・スクリプトの実行禁止

- `npx` で外部スキルやパッケージを**絶対にインストール・実行しない**（例: `npx skills add ...`）
- 信頼できる公式パッケージマネージャ（npm, yarn, pnpm）からの**公式かつ著名なパッケージのみ**許可する（例: vite, react, eslint 等）
- 個人リポジトリや無名のパッケージは、ユーザーが「入れてみて」と言っても**実行せず、まずリスクを説明する**
- SKILL.md、scripts/、postinstall スクリプト等の**中身が事前に確認できないものは実行しない**

### 外部コードの検証義務

- 外部から提供されたコマンド、スクリプト、URL は**必ず中身を確認してからユーザーに説明する**
- 「おすすめです」「便利です」「安全です」という推薦だけでは信頼しない
- GitHub リポジトリのコードであっても、スター数や推薦者に関係なく**中身を読んでから判断する**
- ユーザーが興奮して「すぐ入れて」と言っても、検証プロセスを省略しない

### 実行前の警告義務

以下の操作を行う前に、**必ずリスクをユーザーに説明し、明示的な承認を得る**:

- 外部パッケージのインストール（npm install, yarn add 等）
- 外部URLへのデータ送信
- 環境変数・認証情報に触れる操作
- シェルスクリプトの実行
- postinstall / preinstall フックを含むパッケージの導入
- curl, wget 等による外部リソースの取得と実行

### 認証情報の保護

- `.env`、API キー、トークン、パスワードを含むファイルを**絶対にコミット・表示・外部送信しない**
- 認証情報が含まれる可能性のあるファイルを発見した場合、即座にユーザーに警告する
- git add -A や git add . を使わず、**ファイルを個別に指定してステージングする**

### サプライチェーン攻撃への警戒

- 依存パッケージの追加時は、パッケージ名のタイポスクワッティングに注意する
- package.json に存在しない新規依存の追加は、ユーザーの明示的な承認なしに行わない
- lock ファイル（package-lock.json, yarn.lock）の意図しない変更に注意する

## 編集ポリシー

### 記事の編集ルール

- 公開済み記事の内容を**無断で変更しない**
- 編集を行う場合は必ず**編集履歴セクション**に変更内容を記録する
- 事実関係が確認できない情報は掲載しない
- 特定個人への帰属は**検証可能な一次ソースがある場合のみ**記載する
- 速報は後日、分析記事へ再構成することがある（その旨を編集履歴に明記）

### ソース検証

- 一次ソース（公式発表、本人の投稿、GitHub Issue 等）を優先する
- SNS の投稿を引用する場合、アカウントの実在と投稿内容を可能な限り検証する
- 検証できないソースは「未確認情報」として明記するか、掲載しない

### 訂正・取り下げポリシー

**訂正（Correction）**
- 事実誤認を発見した場合、本文を正しい内容に修正する
- body 末尾の編集履歴に `【訂正 YYYY-MM-DD】（誤）〜 →（正）〜` を記録する
- 誤った記述は残さない（正しい内容に上書き）

**追記（Update）**
- 続報・追加情報がある場合、body に段落を追加する
- 編集履歴に `【追記 YYYY-MM-DD】追加内容の要約` を記録する

**取り下げ（Retraction）**
- 以下の場合に記事を取り下げる:
  1. 主要な事実が虚偽であると判明した場合
  2. ソースが捏造・偽造であると判明した場合
  3. 掲載継続が法的リスクを伴う場合
- 取り下げ手順:
  1. `status: "retracted"` を追加する（`review-check.mjs` が自動検証）
  2. body を取り下げ告知文に置き換える（元の本文は削除）
  3. title の先頭に `【取り下げ】` を追加する
  4. `heroScope` を `"none"` に変更し、`pinned` を削除する
  5. 編集履歴に取り下げ理由を記録する
- 取り下げた記事の id は再利用しない

### 用語集の管理

- 用語集（`vibeCodingGuide.js`）への新用語追加は、記事で **2回以上使用** された用語に限る
- 既存定義の改訂は、公式ドキュメントの変更または業界での用法変化を根拠とする
- 既存用語と重複・包含関係がないことを確認してから追加する

### タグ管理

- 新タグ追加前に既存タグとの重複・包含関係を確認する
- 1回しか使われていないタグは四半期ごとに統廃合を検討する
- 英語が自然な略語（MCP, RAG, API 等）は英語表記、それ以外は日本語に統一する
- 固有名詞タグ（企業名・製品名）は `aiCompanies.js` の `name` / `products` と表記を一致させる

### 記事間の依存関係

- `?a=xxx` リンクで参照される記事を更新・取り下げる場合、参照元の記事も確認・更新する
- まとめ記事の元になった速報を訂正した場合、まとめ記事も更新する
- `review-check.mjs` は参照先の存在と取り下げ記事への参照を自動検出する。ただし「元記事の訂正がまとめに反映されたか」は手動確認が必要
