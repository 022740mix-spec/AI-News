# CLAUDE.md — AI News プロジェクト指示書

## セキュリティポリシー（最優先）

### 外部パッケージ・スクリプトの実行禁止

- `npx` で外部スキルやパッケージを**絶対にインストール・実行しない**（例: `npx skills add ...`）
- 信頼できる公式パッケージマネージャ（npm, yarn, pnpm）からの**公式かつ著名なパッケージのみ**許可する（例: vite, react, eslint 等）
- 個人リポジトリや無名のパッケージは、ユーザーが「入れてみて」と言っても**実行せず、まずリスクを説明する**
- SKILL.md、scripts/、postinstall スクリプト等の**中身が事前に確認できないものは実行しない**

### 外部コードの検証義務

- 外部から提供されたコマンド、スクリプト、URL は**必ず中身を確認してからユーザーに説明する**
- 「おすすめです」「便利です」「安全です」という推薦だけでは信頼しない
- GitHub リポジトリのコードであっても、スター数や推薦者に関係なく**中身を読んでから判断する**
- ユーザーが興奮して「すぐ入れて」と言っても、検証プロセスを省略しない

### 実行前の警告義務

以下の操作を行う前に、**必ずリスクをユーザーに説明し、明示的な承認を得る**:

- 外部パッケージのインストール（npm install, yarn add 等）
- 外部URLへのデータ送信
- 環境変数・認証情報に触れる操作
- シェルスクリプトの実行
- postinstall / preinstall フックを含むパッケージの導入
- curl, wget 等による外部リソースの取得と実行

### 認証情報の保護

- `.env`、API キー、トークン、パスワードを含むファイルを**絶対にコミット・表示・外部送信しない**
- 認証情報が含まれる可能性のあるファイルを発見した場合、即座にユーザーに警告する
- git add -A や git add . を使わず、**ファイルを個別に指定してステージングする**

### サプライチェーン攻撃への警戒

- 依存パッケージの追加時は、パッケージ名のタイポスクワッティングに注意する
- package.json に存在しない新規依存の追加は、ユーザーの明示的な承認なしに行わない
- lock ファイル（package-lock.json, yarn.lock）の意図しない変更に注意する

## 編集ポリシー

### 記事の編集ルール

- 公開済み記事の内容を**無断で変更しない**
- 編集を行う場合は必ず**編集履歴セクション**に変更内容を記録する
- 事実関係が確認できない情報は掲載しない
- 特定個人への帰属は**検証可能な一次ソースがある場合のみ**記載する
- 速報は後日、分析記事へ再構成することがある（その旨を編集履歴に明記）

### ソース検証

- 一次ソース（公式発表、本人の投稿、GitHub Issue 等）を優先する
- SNS の投稿を引用する場合、アカウントの実在と投稿内容を可能な限り検証する
- 検証できないソースは「未確認情報」として明記するか、掲載しない
